# 职能沟通翻译助手

一个面向面试场景的 AI 沟通翻译助手，用来减少产品经理和开发工程师在需求讨论中的理解偏差。

它支持两种翻译方向：

- 产品 -> 开发：把业务描述翻译成技术团队可执行、可评估的表达
- 开发 -> 产品：把技术方案翻译成产品和业务更容易理解的语言

同时支持：

- DeepSeek 模型调用
- SSE 流式输出
- 自动识别输入更像“需求讨论”还是“技术方案”
- 主动补全缺失信息与待确认问题

## 快速开始

### 1. 安装依赖

```bash
pip3 install -r requirements.txt
```

### 2. 配置 API Key

复制一份环境变量模板：

```bash
cp .env.example .env
```

然后在 `.env` 中填写你的 DeepSeek 配置：

```env
DEEPSEEK_API_KEY=your_deepseek_api_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
```

请不要把真实 API Key 提交到仓库。

### 3. 启动项目

```bash
uvicorn app.main:app --reload
```

浏览器访问：

```text
http://127.0.0.1:8000
```

## 功能说明

### 1. 双向沟通翻译

- 产品 -> 开发：保留业务目标，同时补齐算法建议、数据依赖、性能要求、工作量估算
- 开发 -> 产品：保留技术事实，同时解释用户体验影响、业务价值、成本收益和风险

### 2. 自动场景识别

系统会根据输入内容中的关键词，判断它更像：

- 需求讨论
- 技术方案
- 待判断

识别结果只作为提示，不会强制覆盖用户选择的方向。

### 3. 流式展示

后端通过 SSE 持续返回：

- `meta`
- `chunk`
- `done`
- `error`

前端一边实时显示原始流式输出，一边在完成后按标题整理成结构化卡片。

## 测试用例

### 用例 1：产品视角输入

输入：

```text
我们需要一个智能推荐功能，提升用户停留时长
```

期望翻译特点：

- 有清晰的需求目标拆解
- 给出可能的推荐策略
- 说明数据来源和依赖
- 提到性能与实时性要求
- 给出粗粒度工作量评估
- 列出待确认问题

### 用例 2：开发视角输入

输入：

```text
我们优化了数据库查询，QPS提升了30%
```

期望翻译特点：

- 能解释对用户体验的影响
- 能映射到业务增长空间
- 能解释成本和效率收益
- 会提醒风险和限制

## 提示词设计说明

这个项目的关键不是“简单调用模型”，而是让模型真正站在不同角色视角里说话。

### 产品 -> 开发

系统提示词强调：

- 面向开发工程师、技术负责人和研发经理
- 输出必须包含技术实现关注点
- 不允许把推断写成既定事实
- 必须输出固定结构：
  - 需求目标
  - 建议技术路径
  - 数据与依赖
  - 性能与实时性
  - 工作量评估
  - 待确认问题

### 开发 -> 产品

系统提示词强调：

- 面向产品经理和业务同学
- 优先解释用户体验和商业价值
- 避免堆技术术语
- 必须输出固定结构：
  - 技术变更概要
  - 用户体验影响
  - 业务价值
  - 成本与效率收益
  - 风险与限制
  - 建议沟通口径

### 缺失信息补全

两种方向都要求模型：

- 明确区分“已知事实”和“合理推断”
- 在信息不足时显式给出“待确认问题”
- 尽量输出可继续讨论、可落地的内容，而不是空泛描述
